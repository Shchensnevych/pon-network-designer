# Список змін (Changelog)

Усі помітні зміни у цьому проекті будуть документуватися у цьому файлі.
Формат базується на [Keep a Changelog](https://keepachangelog.com/uk/),
а проект використовує [Semantic Versioning (Семантичне Версіювання)](https://semver.org/lang/uk/).

## [1.4.0] - 2026-03-03

### Додано (Added)

- Новий компактний інтерфейс (Вкладки) для модального вікна MDU: перемикання між під'їздами без скролінгу.
- Динамічний лічильник кількості підключених абонентів `(X/Y)` на вкладках MDU.
- Розумне віялове розподілення (Arc Layout) для тултипів MDU та ONU на етапі наближення, để запобігти накладанню тексту тексту.
- Автоматичний розворот тултипів у бік, найближчий до центру карти (Directional Arc).
- Точкові "Сліпі кінці" (Endpoints) на лініях-виносках тултипів для точного візуального прив'язування.
- Ефект напівпрозорого (закритого) скла бекграундів тултипів карти.

### Змінено (Changed)

- Перероблено дизайн карток поверхів в MDU: додано відступи, колір фону та яскраві лівосторонні межі.
- Тултипи тепер автоматично розкриваються тільки при зумі `16+` для уникнення візуального шуму на глобальній карті.

### Виправлено (Fixed)

- Критичний баг збою при розрахунку оптичних сигналів після відкриття MDU-модалки.
- "Circular JSON error", що спричиняв збої під час збереження станів через внутрішні прив'язки Leaflet.
- Зникнення сигналу в MDU через неправильне оброблення непідключених сплітерів.

## [1.3.0] - 2026-03-02

### Додано (Added)

- Інтерактивні підрахунки параметрів абонентів: автоматичний калькулятор `Total Flats` у властивостях MDU.
- Деталізовані шляхи проходження оптичного сигналу (Loss Analysis) у випадаючих списках.

### Змінено (Changed)

- Повністю замінено старі емоджі на стабільні геометричні юнікод-символи (⬢, ⯁, ◼) для всіх елементів інфраструктури, щоб уникнути багів рендерингу на різних ОС.
- Агреговано та розширено текстові скорочення на мапі (Пов. -> Поверх, П. -> Під'їзд).

### Виправлено (Fixed)

- Збій `TypeError Cannot read properties of undefined (reading 'spOut')` під час завантаження збережених проектів.

## [1.2.0] - 2026-02-27

### Додано (Added)

- Новий інструмент/елемент: **Оптична Муфта** (Splice Closure). Тепер це незалежний елемент з окремою іконкою та ізольованим каунтером.
- Логіку зварювальної "Касети" (Splice Cassette) та інтерфейс матриці з'єднань всередині Муфт.
- Детальну Mermaid.js візуалізацію топології для перегляду логічного дерева з'єднань (граф інфраструктури).

### Змінено (Changed)

- Винесено всі CSS стилі з основного HTML-моноліту в окремий файл бази стилів `styles.css`.
- Оновлено документацію по роботі з Муфтами та MDU архітектурою.
- Замінено каркасні кнопки меню (Wireframe) на сучасні суцільні інтерактивні елементи дизайну.

### Виправлено (Fixed)

- Проблему зникнення сигналів при транзиті абонентів (Target Path) до поверхових сплітерів багатоквартирних будинків.
- Баг з подвоєними заголовками модальних вікон.

## [1.1.0] - 2026-02-23

### Додано (Added)

- Інтелектуальну систему Tooltips (підписів об'єктів) на карті та алгоритми відстежування оптичних шляхів.
- Аналізатор ємності магістралей на кабелях (наприклад, "150 м (12 жил)").
- Вбудовану функцію Auto-Transit для автоматичного прокладання волокон від порту OLT через мультиплексори/сплітери.
- Кольорову диференціацію кабелів залежно від підключеного порту на OLT.
- Великий Звіт Проекту (Optical Loss budget, OLT port loading, Economic cost estimation).

### Виправлено (Fixed)

- Лаги прокрутки у модальних вікнах (прибрано тяжкий ефект backdrop-filter).
- Баг 0-го індексу, коли `Core 1` визначалося алгоритмом обходу як пустота (`falsy value`) і з'єднання втрачалося.
